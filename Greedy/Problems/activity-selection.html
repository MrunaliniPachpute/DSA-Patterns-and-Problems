Activity Selection

Problem Statement

Given start and finish times of activities, select the maximum number of activities that can be performed by a single person assuming a person can only work on one activity at a time.

Example: Input: start = [1, 3, 0, 5, 8, 5], end = [2, 4, 6, 7, 9, 9]
Output: 4 (Activities chosen: {1,2}, {3,4}, {5,7}, {8,9})

Brute Force Approach

Try all subsets of activities to find maximum non-overlapping set.

Time Complexity: O(2^n), Space Complexity: O(1)

generate all subsets of activities
for each subset:
    check if all activities are non-overlapping
return maximum count

Better Approach

Sort activities by start time and use recursion to select compatible activities.

Time Complexity: O(nÂ²), Space Complexity: O(n)

Optimal Approach (Greedy)

Sort by finish time and pick the next activity that starts after the last selected one finishes.

Time Complexity: O(n log n), Space Complexity: O(1)

sort activities by finish time
select first activity
for each next activity:
    if start >= last_finish:
        select activity
return count
