Activity Selection

Problem Statement

Given start and finish times of activities, select the maximum number of activities that can be performed by a single person assuming a person can only work on one activity at a time.

Example:
Input: start = [1, 3, 0, 5, 8, 5], end = [2, 4, 6, 7, 9, 9]
Output: 4 (Activities chosen: {1,2}, {3,4}, {5,7}, {8,9})

Approach 1: Greedy by Finishing Time

Explanation: Sort activities by their finishing times. Pick the first activity, then repeatedly pick the next activity that starts after the last chosen one finishes.

Time Complexity: O(n log n) (due to sorting)

Space Complexity: O(1)

sort activities by finish time
choose first activity
for each next activity:
    if start >= last_finish:
        select activity
return count

Approach 2: Using Priority Queue

Explanation: Insert activities in a min-heap based on finish time. Keep popping activities that overlap with the last selected one.

Time Complexity: O(n log n)

Space Complexity: O(n)

push all activities (start, finish) into min-heap by finish time
take first activity
while heap not empty:
    pop activity
    if start >= last_finish:
        select it
return count
