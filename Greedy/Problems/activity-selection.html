Activity Selection Problem

Problem Statement

Given start and finish times of activities, select the maximum number of activities that can be performed by a single person assuming a person can only work on one activity at a time.

Example: start = [1,3,0,5,8,5], end = [2,4,6,7,9,9] â†’ Output: 4

Brute Force Approach

Try all possible subsets of activities and check which set is valid (non-overlapping). Return the size of the maximum valid subset.

for each subset of activities:
    check if no two activities overlap
    if valid:
        update maximum
return maximum

Better Approach

Sort activities by start time. Then, for each activity, recursively try to include it if it does not overlap with the previously included one. This reduces redundant subsets but still takes quadratic time.

sort activities by start time
function recur(index, last_end):
    if index == n:
        return 0
    skip = recur(index+1, last_end)
    take = 0
    if start[index] >= last_end:
        take = 1 + recur(index+1, end[index])
    return max(skip, take)

Optimal Approach (Greedy)

Sort by finish time. Pick the activity that finishes earliest, then repeatedly pick the next activity that starts after the last chosen one finishes.

sort activities by finish time
select first activity
last_end = finish of first
for each activity in sorted list:
    if start >= last_end:
        select it
        last_end = finish
