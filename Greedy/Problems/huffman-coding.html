Huffman Coding

Problem Statement

Given characters and their frequencies, create a prefix-free binary code (Huffman code) that minimizes total weighted path length.

Example: Characters: ['a', 'b', 'c', 'd'], Frequencies: [5, 9, 12, 13], Output: Minimum encoding tree

Brute Force Approach

Try all possible prefix codes to find minimal weighted sum.

Time Complexity: Exponential, Space Complexity: Very large

Better Approach

Use DP to compute optimal binary trees (still costly).

Time Complexity: O(n³), Space Complexity: O(n²)

Optimal Approach (Greedy)

Use a min-heap: repeatedly merge the two smallest frequency nodes until only one tree remains.

Time Complexity: O(n log n), Space Complexity: O(n)

create min-heap with all characters by frequency
while heap size > 1:
    extract two smallest nodes
    create new internal node with sum frequency
    insert back into heap
return remaining node as root
