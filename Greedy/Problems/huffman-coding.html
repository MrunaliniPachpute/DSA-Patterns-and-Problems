Huffman Encoding

Problem Statement

Given frequencies of characters, build a Huffman Tree and encode the characters to minimize the total cost of encoding.

Example:
Characters = {a:5, b:9, c:12, d:13, e:16, f:45}
Output Codes may be:
f=0, c=100, d=101, a=1100, b=1101, e=111

Approach: Greedy using Min Heap

Explanation: Push all characters with frequency into min-heap. Extract two minimums, create a new node with their sum, push back into heap. Repeat until one node remains.

Time Complexity: O(n log n)

Space Complexity: O(n)

create min-heap of all chars with freq
while heap size > 1:
    left = extract min
    right = extract min
    create new node with freq = left+right
    push new node into heap
assign 0/1 to edges and traverse tree for codes
