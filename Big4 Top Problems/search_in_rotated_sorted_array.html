Search in Rotated Sorted Array

Medium

There is an integer array `nums` sorted in ascending order (with distinct values). Prior to being passed to your function, `nums` is **rotated** at an unknown pivot index $k$, such that the resulting array is $\left[\text{nums}[k], \text{nums}[k+1], \dots, \text{nums}[n-1], \text{nums}[0], \dots, \text{nums}[k-1]\right]$.

Given the rotated array `nums` and an integer `target`, return the index of `target` if it is in `nums`, or `-1` if it is not.

You must write an algorithm with $O(\log n)$ runtime complexity.

Example 1:

Input: nums = [4, 5, 6, 7, 0, 1, 2], target = 0 Output: 4

Example 2:

Input: nums = [4, 5, 6, 7, 0, 1, 2], target = 3 Output: -1

Constraints:

  • $1 \le \text{nums.length} \le 5000$
  • $-10^4 \le \text{nums}[i], \text{target} \le 10^4$

Approach: Modified Binary Search

Since the array is partially sorted and we require $O(\log n)$ complexity, we must use **Binary Search**. The key modification is determining which half of the array is currently sorted and then checking if the `target` falls within that sorted range.

In each step of the binary search, one half of the array will always be correctly sorted (the rotation point lies in the other half). We use this sorted half to decide where to continue the search.

Steps:

  1. Initialize `left = 0` and `right = n - 1`.
  2. While `left <= right`:
    • Calculate `mid`.
    • If `nums[mid] == target`, return `mid`.
    • **Determine the Sorted Half:**
      • **Case 1: Left Half is Sorted** (i.e., `nums[left] <= nums[mid]`):
        • Check if `target` is within this sorted range: `nums[left] <= target < nums[mid]`. If yes, set `right = mid - 1`.
        • Otherwise, the target must be in the right (unsorted) half: set `left = mid + 1`.
      • **Case 2: Right Half is Sorted** (i.e., `nums[left] > nums[mid]`):
        • Check if `target` is within this sorted range: `nums[mid] < target <= nums[right]`. If yes, set `left = mid + 1`.
        • Otherwise, the target must be in the left (unsorted) half: set `right = mid - 1`.
  3. If the loop finishes without finding the target, return `-1`.

Example Solution (Conceptual Python):

def search(nums, target):
    left, right = 0, len(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2

        if nums[mid] == target:
            return mid

        # Check if the left half is sorted
        if nums[left] <= nums[mid]:
            # Target is in the left sorted half
            if nums[left] <= target and target < nums[mid]:
                right = mid - 1
            else:
                # Target is in the right unsorted half
                left = mid + 1
        
        # Otherwise, the right half is sorted
        else:
            # Target is in the right sorted half
            if nums[mid] < target and target <= nums[right]:
                left = mid + 1
            else:
                # Target is in the left unsorted half
                right = mid - 1

    return -1
