Longest Common Subsequence (LCS)

Medium

Given two strings `text1` and `text2`, return the length of their **longest common subsequence**.

A **subsequence** of a string is a new string generated from the original string with some characters deleted (or none) without changing the relative order of the remaining characters.

A **common subsequence** is a subsequence that is common to both strings.

Example 1:

Input: text1 = "abcde", text2 = "ace" Output: 3 Explanation: The longest common subsequence is "ace" and its length is 3.

Example 2:

Input: text1 = "abc", text2 = "abc" Output: 3

Example 3:

Input: text1 = "abc", text2 = "def" Output: 0

Constraints:

  • $1 \le \text{text1.length}, \text{text2.length} \le 1000$
  • The strings consist of lowercase English characters.

Approach: 2D Dynamic Programming

This problem exhibits **optimal substructure** and **overlapping subproblems**, making it an ideal candidate for **Dynamic Programming (DP)**.

We use a 2D DP array, `dp[i][j]`, to store the length of the LCS of the prefixes `text1[0...i-1]` and `text2[0...j-1]`.

The transition logic depends on whether the characters at the current indices match:

Recurrence Relation:

  1. **If `text1[i-1] == text2[j-1]` (Match):**

    The LCS length increases by 1 based on the LCS of the strings excluding these matched characters.

    $$\mathbf{DP}[i][j] = 1 + \mathbf{DP}[i-1][j-1]$$
  2. **If `text1[i-1] != text2[j-1]` (Mismatch):**

    The LCS length is the maximum of: (1) LCS of `text1` excluding its last character, and (2) LCS of `text2` excluding its last character.

    $$\mathbf{DP}[i][j] = \max(\mathbf{DP}[i-1][j], \mathbf{DP}[i][j-1])$$

Time Complexity: $O(M \times N)$, where $M$ and $N$ are the lengths of `text1` and `text2`. The solution involves filling an $M \times N$ DP table once.

Example Solution (Conceptual Python):

def longestCommonSubsequence(text1, text2):
    m, n = len(text1), len(text2)
    # Initialize DP table of size (m+1) x (n+1) with zeros
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            
            # Check characters in the original strings (indices i-1 and j-1)
            if text1[i - 1] == text2[j - 1]:
                # Characters match: add 1 to the diagonal element
                dp[i][j] = 1 + dp[i - 1][j - 1]
            else:
                # Characters do not match: take the max of the top or left element
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    # The result is stored at the bottom-right corner
    return dp[m][n]
