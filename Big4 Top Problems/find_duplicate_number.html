Find Duplicate Number

Medium

Given an array of integers `nums` containing $n+1$ integers where each integer is between 1 and $n$ (inclusive). There is only **one repeated number** in `nums`. Find and return this repeated number.

You must solve this problem **without modifying the array `nums`** and must use only constant $O(1)$ extra space.

Example 1:

Input: nums = [1, 3, 4, 2, 2] Output: 2

Example 2:

Input: nums = [3, 1, 3, 4, 2] Output: 3

Constraints:

  • $1 \le n \le 10^5$
  • $\text{nums.length} == n + 1$
  • $1 \le \text{nums}[i] \le n$
  • All the integers in `nums` appear only once except for precisely one integer which appears two or more times.

Optimal Approach: Tortoise and Hare (Floyd's Cycle Detection)

This solution leverages the constraints of the problem to model the array as a **Linked List** with a cycle, where the duplicate number is the **entry point of the cycle**.

Since all numbers are between 1 and $n$, and the array size is $n+1$, if we treat `i` as a node and `nums[i]` as a pointer to the next node, the repeated number creates a **cycle** in this conceptual linked list.

The steps are identical to finding the start of a cycle in a linked list:

Time Complexity: $O(N)$ (linear time). **Space Complexity:** $O(1)$ (constant extra space).

Steps:

  1. **Phase 1: Detect the Cycle:**
    • Initialize two pointers, `slow` and `fast`, starting at index 0.
    • The `slow` pointer moves one step at a time: `slow = nums[slow]`.
    • The `fast` pointer moves two steps at a time: `fast = nums[nums[fast]]`.
    • Continue until `slow` and `fast` meet, confirming the presence of a cycle.
  2. **Phase 2: Find the Cycle Start (The Duplicate):**
    • Reset the `slow` pointer back to the start (index 0).
    • Move both `slow` and `fast` pointers one step at a time.
    • The point where they meet again is the **entry point of the cycle**, which is the **duplicate number**.

Example Solution (Conceptual Python):

def findDuplicate(nums):
    # Phase 1: Detect the intersection point of slow and fast pointers
    slow = nums[0]
    fast = nums[nums[0]]

    while slow != fast:
        slow = nums[slow]
        fast = nums[nums[fast]]

    # Phase 2: Find the entrance to the cycle (the duplicate)
    # Reset slow to the start
    slow = nums[0]
    
    # Move both at the same pace until they meet again
    while slow != fast:
        slow = nums[slow]
        fast = nums[fast]

    # The intersection point is the duplicate number
    return slow
