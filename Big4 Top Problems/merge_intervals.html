Merge Overlapping Intervals

Medium

Given an array of intervals, where each interval `intervals[i] = [start_i, end_i]`, merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the intervals in the input.

Example 1:

Input: intervals = [[1, 3], [2, 6], [8, 10], [15, 18]] Output: [[1, 6], [8, 10], [15, 18]] Explanation: Intervals [1, 3] and [2, 6] overlap, so merge them into [1, 6].

Example 2:

Input: intervals = [[1, 4], [4, 5]] Output: [[1, 5]] Explanation: Intervals [1, 4] and [4, 5] are considered overlapping as they meet at 4.

Constraints:

  • $1 \le \text{intervals.length} \le 10^4$
  • $\text{intervals}[i].length == 2$
  • $0 \le \text{start}_i \le \text{end}_i \le 10^4$

Approach: Sorting and Linear Scan (Greedy)

The key to solving this efficiently is to first **sort the intervals based on their start times**. Once sorted, merging can be done with a single linear pass (a **Greedy** approach).

By sorting, we guarantee that for any two consecutive intervals, if they overlap, the first interval's start time is less than or equal to the second interval's start time. We only need to check if the current interval's start time is less than or equal to the end time of the *last merged interval*.

Time Complexity: $O(N \log N)$ due to the initial sort, where $N$ is the number of intervals. The linear scan takes $O(N)$. **Space Complexity:** $O(N)$ to store the result.

Steps:

  1. **Sort** the `intervals` array based on the starting value of each interval.
  2. Initialize a `merged` list with the first interval.
  3. Iterate through the remaining intervals, starting from the second one:
    • Let the `last_end` be the end time of the last interval in the `merged` list.
    • Let the `current_start` be the start time of the current interval being examined.
    • **If `current_start <= last_end` (Overlap):** Merge them by updating the `last_end` in the `merged` list to $\max(\text{last\_end}, \text{current\_end})$.
    • **If `current_start > last_end` (No Overlap):** The current interval starts a new non-overlapping segment. Add the current interval to the `merged` list.
  4. Return the `merged` list.

Example Solution (Conceptual Python):

def merge(intervals):
    if not intervals:
        return []

    # 1. Sort the intervals based on the start time
    intervals.sort(key=lambda x: x[0])
    
    merged = []
    # Initialize merged list with the first interval
    merged.append(intervals[0]) 

    for start, end in intervals[1:]:
        # Get the end time of the last merged interval
        last_merged_end = merged[-1][1]
        
        # Check for overlap
        if start <= last_merged_end:
            # Overlap: Extend the end time of the last merged interval
            # by taking the maximum of the current end and the last merged end
            merged[-1][1] = max(last_merged_end, end)
        else:
            # No overlap: Add the current interval as a new one
            merged.append([start, end])
            
    return merged
