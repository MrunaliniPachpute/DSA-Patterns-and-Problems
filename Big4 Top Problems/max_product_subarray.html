Maximum Product Subarray

Medium

Given an integer array `nums`, find a contiguous non-empty subarray within the array that has the largest product, and return the product.

The test cases are generated so that the answer will fit in a 32-bit integer.

A **subarray** is a contiguous part of an array.

Example 1:

Input: nums = [2, 3, -2, 4] Output: 6 Explanation: The subarray [2, 3] has the largest product 6.

Example 2:

Input: nums = [-2, 0, -1] Output: 0 Explanation: The result cannot be 2, because [-2, -1] is not a subarray. The largest product is 0 (from subarray [0] or [-2, 0] or [0, -1]).

Constraints:

  • $1 \le \text{nums.length} \le 2 \times 10^4$
  • $-10 \le \text{nums}[i] \le 10$

Approach: Dynamic Programming (Tracking Min and Max Products)

This problem is similar to the Maximum Subarray Sum (Kadane's), but requires more complex logic due to the presence of **negative numbers**. A small negative number multiplied by another negative number can yield a large positive product.

Therefore, at each step, we must keep track of two values ending at the current position `i`:

  • **`current_max`**: The maximum product subarray ending at `i`.
  • **`current_min`**: The minimum product subarray ending at `i`.

When we encounter a negative number, `current_max` and `current_min` effectively swap roles, as multiplying a large positive number by a negative number gives a small negative, and multiplying a large negative number by a negative number gives a large positive.

Time Complexity: $O(N)$ (linear time). **Space Complexity:** $O(1)$ (constant extra space).

Steps:

  1. Initialize `overall_max`, `current_max`, and `current_min` all to `nums[0]`.
  2. Iterate through the array starting from the second element (`i = 1`).
  3. For each element `num = nums[i]`:
    • Calculate the potential products: `p1 = num * current_max` and `p2 = num * current_min`.
    • Update `current_max` to be the maximum of: `num`, `p1`, and `p2`.
    • Update `current_min` to be the minimum of: `num`, `p1`, and `p2`. (We must use the *old* `current_max` to calculate `p1` and `p2` before updating `current_max`).
    • Update `overall_max = max(overall_max, current_max)`.
  4. Return `overall_max`.

Example Solution (Conceptual Python):

def maxProduct(nums):
    if not nums:
        return 0

    # Initialize current_max, current_min, and overall_max with the first element
    current_max = nums[0]
    current_min = nums[0]
    overall_max = nums[0]

    for i in range(1, len(nums)):
        num = nums[i]

        # temp_max is required because current_max is needed for current_min calculation 
        # BEFORE current_max itself is updated.
        temp_max = current_max 
        
        # New max product ending at i can be:
        # 1. Just the current number
        # 2. current_max * current number
        # 3. current_min * current number (important for negative numbers)
        current_max = max(num, temp_max * num, current_min * num)
        
        # New min product ending at i can be:
        current_min = min(num, temp_max * num, current_min * num)
        
        # Update the overall maximum found so far
        overall_max = max(overall_max, current_max)

    return overall_max
